# دليل المساهمة في المشروع

نرحب بمساهماتكم في تطوير نظام إدارة الطلبات! هذا الدليل سيساعدكم على المساهمة بشكل فعال.

## كيفية المساهمة

### 1. إعداد البيئة المحلية

```bash
# استنساخ المشروع
git clone <repository-url>
cd orders_sys

# تثبيت التبعيات
npm install

# إعداد متغيرات البيئة
cp env.example .env
# قم بتحديث ملف .env بالقيم الصحيحة

# تشغيل المشروع
npm run dev
```

### 2. إنشاء فرع جديد

```bash
# إنشاء فرع جديد للميزة
git checkout -b feature/your-feature-name

# أو لإصلاح مشكلة
git checkout -b fix/issue-description
```

### 3. قواعد التطوير

#### أسماء الفروع
- `feature/` للميزات الجديدة
- `fix/` لإصلاح المشاكل
- `docs/` للوثائق
- `refactor/` لإعادة هيكلة الكود
- `test/` للاختبارات

#### رسائل Commit
استخدم التنسيق التالي:
```
<type>(<scope>): <subject>

<body>

<footer>
```

**الأمثلة:**
- `feat(orders): add bulk delete functionality`
- `fix(products): resolve image upload issue`
- `docs(readme): update installation instructions`
- `style(ui): improve responsive design`

#### أنواع التغييرات
- `feat`: ميزة جديدة
- `fix`: إصلاح مشكلة
- `docs`: تغييرات في الوثائق
- `style`: تغييرات في التصميم
- `refactor`: إعادة هيكلة الكود
- `perf`: تحسين الأداء
- `test`: إضافة أو تعديل الاختبارات
- `chore`: تغييرات في الأدوات أو البناء

### 4. معايير الكود

#### TypeScript
- استخدم TypeScript بدلاً من JavaScript
- حدد أنواع البيانات بوضوح
- استخدم interfaces للكائنات المعقدة

#### React
- استخدم Functional Components مع Hooks
- استخدم TypeScript للـ props
- اجعل المكونات قابلة لإعادة الاستخدام

#### CSS
- استخدم Tailwind CSS للتصميم
- اجعل التصميم متجاوب
- استخدم الألوان من نظام التصميم

#### الأسماء
- استخدم أسماء واضحة ومفهومة
- استخدم اللغة الإنجليزية للمتغيرات والدوال
- استخدم اللغة العربية للنصوص المعروضة للمستخدم

### 5. الاختبار

```bash
# تشغيل الاختبارات
npm run test

# فحص جودة الكود
npm run lint

# إصلاح مشاكل الكود
npm run lint:fix
```

### 6. إرسال التغييرات

```bash
# إضافة التغييرات
git add .

# إنشاء commit
git commit -m "feat(orders): add bulk delete functionality"

# رفع التغييرات
git push origin feature/your-feature-name
```

### 7. إنشاء Pull Request

1. اذهب إلى صفحة المشروع على GitHub
2. اضغط على "New Pull Request"
3. اختر الفرع الخاص بك
4. اكتب وصفاً واضحاً للتغييرات
5. أضف لقطات شاشة إذا لزم الأمر
6. اطلب مراجعة من المطورين

## إرشادات خاصة

### إضافة ميزات جديدة
- تأكد من أن الميزة مطلوبة ومفيدة
- اكتب وثائق واضحة للميزة
- أضف اختبارات للميزة الجديدة
- اجعل التصميم متجاوب

### إصلاح المشاكل
- تأكد من فهم المشكلة جيداً
- اختبر الحل قبل الإرسال
- أضف اختبارات لمنع تكرار المشكلة

### تحسين الأداء
- قم بقياس الأداء قبل وبعد التحسين
- تأكد من عدم كسر الوظائف الموجودة
- وثق التحسينات

### التصميم المتجاوب
- اختبر على أجهزة مختلفة
- استخدم Tailwind CSS classes المناسبة
- تأكد من سهولة الاستخدام على الهواتف

## المساعدة والدعم

إذا واجهت أي مشاكل:
1. تحقق من الوثائق الموجودة
2. ابحث في Issues السابقة
3. أنشئ Issue جديد مع تفاصيل المشكلة
4. تواصل مع فريق التطوير

## شكراً لمساهمتكم! 🙏

نقدر وقتكم وجهدكم في تطوير هذا المشروع. كل مساهمة تساعد في جعل النظام أفضل للمستخدمين.
