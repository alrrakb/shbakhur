# دليل النشر والتشغيل

## 🚀 تشغيل النظام محلياً

### 1. متطلبات التشغيل
- Node.js 18+ 
- npm أو yarn
- حساب Supabase

### 2. تثبيت التبعيات
```bash
npm install
```

### 3. إعداد متغيرات البيئة
تم إنشاء ملف `.env` من `env.example` مع بيانات Supabase الخاصة بك.

### 4. إعداد قاعدة البيانات
1. انتقل إلى [Supabase Dashboard](https://supabase.com/dashboard)
2. اختر مشروعك
3. انتقل إلى SQL Editor
4. انسخ محتوى ملف `database-schema.sql`
5. نفذ الاستعلامات

### 5. تشغيل النظام
```bash
npm run dev
```

سيفتح النظام على `http://localhost:3000`

## 🔐 نظام المصادقة

### إعداد المصادقة في Supabase
1. انتقل إلى Authentication > Settings
2. فعّل Email Authentication
3. أضف مستخدمين جدد من Authentication > Users

### بيانات تسجيل الدخول التجريبية
يمكنك إنشاء مستخدم جديد من واجهة Supabase أو استخدام:
- البريد الإلكتروني: admin@example.com
- كلمة المرور: password123

## 🛠️ استكشاف الأخطاء

### مشاكل شائعة

#### 1. خطأ في الاتصال بـ Supabase
```
Error: Missing Supabase environment variables
```
**الحل:** تأكد من وجود ملف `.env` مع البيانات الصحيحة

#### 2. خطأ في قاعدة البيانات
```
Error: relation "customers" does not exist
```
**الحل:** نفذ استعلامات `database-schema.sql` في Supabase

#### 3. خطأ في المصادقة
```
Error: Invalid login credentials
```
**الحل:** تأكد من إنشاء مستخدم في Supabase Authentication

### سجلات الأخطاء
- افتح Developer Tools (F12)
- انتقل إلى Console لرؤية الأخطاء
- انتقل إلى Network لمراقبة طلبات API

## 📱 النشر على الإنترنت

### النشر على Vercel
1. اربط المشروع بـ GitHub
2. اربط GitHub بـ Vercel
3. أضف متغيرات البيئة في Vercel
4. انشر المشروع

### متغيرات البيئة المطلوبة
```
VITE_SUPABASE_URL=your_supabase_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
VITE_APP_NAME=نظام إدارة الطلبات
VITE_APP_VERSION=1.0.0
```
